# 4. Phase 1: 基础设施详细设计

## 4.1 DBC 链改造 / L2 层

### 4.1.1 改造方案

**问题分析:**
- DBC 链没有原生稳定币
- 需要高性能交易撮合 (<100ms)
- 需要支持跨链资产

**推荐方案: 独立 L2 + DBC 状态存证**
- 独立运行高性能 L2 交易链
- 定期将状态根提交到 DBC 链存证
- DBC 作为争议仲裁层
- 优点: 灵活、可快速启动

### 4.1.2 L2 技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| 共识 | HotStuff BFT | Hyperliquid 同款 |
| 语言 | Rust | 高性能、安全 |
| 存储 | RocksDB | 高性能 KV |
| DA 层 | Celestia | 数据可用性 |
| 参考 | Aptos Core | 开源可用 |

### 4.1.3 HotStuff 共识参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 出块时间 | 100ms | 高频交易需求 |
| 最终性 | 2-3 区块 | 200-300ms 确认 |
| 验证者 | 4-21 个 | 初期 4 个 |
| 容错 | 1/3 | 拜占庭容错 |

### 4.1.4 撮合引擎

**核心特点:**
- 全链上订单簿 (CLOB)
- 内存撮合，纳秒级延迟
- 价格-时间优先
- 支持批量订单

**性能目标:**

| 指标 | 目标 |
|------|------|
| 单市场 TPS | 50,000+ |
| 撮合延迟 | < 1ms |
| 总 TPS | 100,000+ |

### 4.1.5 清算引擎

**保证金率计算:**
```
账户权益 = 余额 + 未实现盈亏
保证金率 = 账户权益 / 维持保证金
保证金率 < 100% → 触发清算
```

**维持保证金率:**

| 仓位价值 | 维持保证金率 | 最高杠杆 |
|----------|--------------|----------|
| < $100K | 0.5% | 200x |
| $100K-$1M | 1% | 100x |
| $1M-$10M | 2% | 50x |
| > $10M | 5% | 20x |

**清算流程:**
1. 渐进式清算 (每次 25%)
2. 以 Oracle 价格为基准
3. 保险基金承担亏损
4. 保险基金不足 → ADL

## 4.3 交易所核心功能

### 4.3.1 永续合约

| 交易对 | 最高杠杆 | 最小单位 |
|--------|----------|----------|
| BTC-PERP | 500x | 0.001 BTC |
| ETH-PERP | 500x | 0.01 ETH |
| SOL-PERP | 200x | 0.1 SOL |
| 其他主流 | 100-200x | varies |
| 山寨/新币 | 20-50x | varies |

**保证金模式:**
- 逐仓保证金 (Isolated) - Phase 1
- 全仓保证金 (Cross) - Phase 1
- 组合保证金 (Portfolio Margin) - Phase 2

**资金费率:**
- 结算: 每 8 小时
- 上下限: ±0.75% / 8h

### 4.3.2 订单类型

**Phase 1:**
- Market (市价单)
- Limit (限价单)
- Stop Market (触发市价单)
- Stop Limit (触发限价单)
- Take Profit (止盈单)
- Stop Loss (止损单)
- Post Only (只做 Maker)
- Reduce Only (只减仓)
- IOC (Immediate or Cancel)
- FOK (Fill or Kill)
- GTC (Good Till Cancel)

**Phase 2:**
- Trailing Stop (追踪止损)
- OCO (二选一)
- TWAP (时间加权平均)
- Iceberg (冰山单)
- VWAP (成交量加权平均)

### 4.3.3 现货交易

- **零手续费** (Maker 0%, Taker 0%)
- 即时结算
- 与合约共享保证金 (Portfolio Margin 模式)

### 4.3.4 合约手续费

| 角色 | 费率 | 对比 Hyperliquid |
|------|------|------------------|
| Maker | 0.01% | 50% 折扣 (原 0.02%) |
| Taker | 0.025% | 50% 折扣 (原 0.05%) |

**手续费分配：**
- 80% → BLP 金库 / 平台
- 20% → 推荐返佣池

## 8.4 Bot 推荐返佣系统

**核心理念：Bot 推荐 Bot，形成 Bot 生态网络**

### 8.4.1 手续费结构

HypeClaw 采用差异化手续费：

| 交易类型 | Maker 费率 | Taker 费率 | 说明 |
|----------|------------|------------|------|
| **现货交易** | 0% | 0% | 完全免费 |
| **合约交易** | 0.01% | 0.025% | Hyperliquid 的 50% |

*对比 Hyperliquid: Maker 0.02%, Taker 0.05%*

### 8.4.2 机制说明

每个用户都有自己的 OpenClaw Bot，Bot 之间可以互相交互和推荐。

当 **Bot A 推荐 Bot B 来 HypeClaw 交易** 时，Bot A 从 Bot B 的**合约交易手续费**中获得返佣。

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Bot 推荐返佣流程                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   Bot A 已在 HypeClaw 交易                                          │
│         │                                                            │
│         │  Bot A 通过 OpenClaw 网络推荐 Bot B:                       │
│         │  "HypeClaw 合约手续费只有 Hyperliquid 的一半，来试试"      │
│         │                                                            │
│         ▼                                                            │
│   Bot B 通过 Bot A 的推荐链接注册 HypeClaw                           │
│         │                                                            │
│         │  Bot B 进行合约交易，产生手续费                            │
│         │                                                            │
│         ▼                                                            │
│   Bot A 获得返佣 (Bot B 合约交易手续费的 20%)                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 8.4.3 返佣计算示例

```
Bot B 月合约交易量: $1,000,000
平均 Taker 交易，手续费率: 0.025%
产生手续费: $1,000,000 × 0.025% = $250

Bot A 返佣 (20%): $250 × 20% = $50/月
```

### 8.4.4 返佣规则

| 场景 | 返佣接收方 | 返佣比例 | 有效期 |
|------|------------|----------|--------|
| Bot A 推荐 Bot B 注册 | Bot A | 20% | 永久 |
| Bot A 推荐 Bot C 注册 | Bot A | 20% | 永久 |
| 同一 Bot 被多个 Bot 推荐 | 首个推荐的 Bot | 20% | 永久 |

**注意：** 返佣仅来自**合约交易手续费**，现货交易无手续费故无返佣。

### 8.4.5 返佣流向示意

```
Bot B 合约交易产生手续费 $100
         │
         ├─► 推荐方 Bot A: $20 (20%)
         │
         └─► 平台/BLP 金库: $80 (80%)
```

### 8.4.6 推荐等级

| 等级 | 累计推荐 Bot 数 | 返佣比例 | 额外奖励 |
|------|-----------------|----------|----------|
| 🥉 Bronze | 1-10 | 20% | - |
| 🥈 Silver | 11-50 | 25% | 推荐位加权 |
| 🥇 Gold | 51-200 | 30% | 官方推荐 + 月度奖金 |
| 💎 Diamond | 200+ | 35% | 首页展示 + 独家活动 |

### 8.4.7 Bot 之间的交互方式

基于 OpenClaw 的多 agent 架构，Bot 之间可以：

1. **直接消息推荐**
```
Bot A → Bot B: "我在 HypeClaw 做合约，手续费很低，
                用我的链接注册：hypeclaw.io/ref/bot_a"
```

### 8.4.8 防作弊机制

- **自推荐禁止**: 同一用户的多个 Bot 之间推荐无效

# 10. API 系统详解

## 10.1 API 总览

```
┌────────────────────────────────────────────────────────────────────────┐
│  API 类型：                                                            │
│                                                                        │
│  1. Public API（公开）                                                 │
│     • 无需认证                                                         │
│     • 行情数据、市场信息                                               │
│                                                                        │
│  2. Bot API（Bot 专用）                                                │
│     • 需要 Bot API Key + 用户授权                                      │
│     • 交易操作、账户查询                                               │
│                                                                        │
│  3. User API（用户专用）                                               │
│     • 需要钱包签名认证                                                 │
│     • 充提、授权管理                                                   │
│                                                                        │
│  4. MoonBot Compatible API（兼容层）                                   │
│     • 兼容 MoonBot API 格式                                            │
│     • 自动转换                                                         │
│                                                                        │
│  协议：REST (HTTPS) + WebSocket (WSS)                                  │
│  数据格式：JSON                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

## 10.2 Public API

**市场数据：**
```
GET  /api/v1/markets                    获取所有市场列表
GET  /api/v1/markets/{market}           获取市场详情
GET  /api/v1/ticker/{market}            获取最新价格
GET  /api/v1/tickers                    获取所有价格
GET  /api/v1/orderbook/{market}         获取订单簿
GET  /api/v1/trades/{market}            获取最新成交
GET  /api/v1/klines/{market}            获取K线数据
GET  /api/v1/funding-rate/{market}      获取资金费率
```

**WebSocket 订阅：**
```
WS /ws/public

// 订阅 ticker
{"op": "subscribe", "channel": "ticker", "market": "BTC-PERP"}

// 订阅订单簿
{"op": "subscribe", "channel": "orderbook", "market": "BTC-PERP"}

// 订阅成交
{"op": "subscribe", "channel": "trades", "market": "BTC-PERP"}
```

## 10.3 Bot API

**认证方式：**
```
Header:
  X-Bot-Api-Key: {bot_api_key}
  X-User-Address: {user_address}      // 被授权的用户地址
  X-Timestamp: {timestamp}
  X-Signature: {signature}            // HMAC-SHA256 签名
```

**交易接口：**
```
POST   /api/v1/bot/order                 下单
DELETE /api/v1/bot/order/{order_id}      取消订单
POST   /api/v1/bot/orders/batch          批量下单
DELETE /api/v1/bot/orders/batch          批量取消
POST   /api/v1/bot/position/close        平仓
PUT    /api/v1/bot/position/leverage     调整杠杆
PUT    /api/v1/bot/position/margin       调整保证金
```

**查询接口：**
```
GET    /api/v1/bot/account               获取账户信息
GET    /api/v1/bot/positions             获取持仓
GET    /api/v1/bot/orders                获取挂单
GET    /api/v1/bot/orders/history        获取订单历史
GET    /api/v1/bot/trades                获取成交历史
GET    /api/v1/bot/permissions           获取授权信息
```

**下单示例：**
```json
// 请求 POST /api/v1/bot/order
{
  "market": "BTC-PERP",
  "side": "buy",
  "type": "limit",
  "price": "50000",
  "size": "0.1",
  "leverage": 50,
  "reduce_only": false,
  "post_only": false,
  "time_in_force": "GTC",
  "client_order_id": "bot_001_12345"
}

// 响应
{
  "success": true,
  "order_id": "1234567890",
  "client_order_id": "bot_001_12345",
  "status": "open",
  "filled_size": "0",
  "remaining_size": "0.1",
  "created_at": 1706745600000
}
```

**WebSocket (Bot 私有频道)：**
```
WS /ws/bot?api_key={api_key}&user={user_address}

// 订阅订单更新
{"op": "subscribe", "channel": "orders"}

// 订阅持仓更新
{"op": "subscribe", "channel": "positions"}

// 订阅账户更新
{"op": "subscribe", "channel": "account"}
```

## 10.4 User API

**认证方式：钱包签名**
```
Header:
  X-User-Address: {user_address}
  X-Timestamp: {timestamp}
  X-Signature: {wallet_signature}    // EIP-712 签名
```

**资产管理：**
```
GET    /api/v1/user/balances             获取余额
POST   /api/v1/user/deposit/address      获取充值地址
POST   /api/v1/user/withdraw             发起提现
GET    /api/v1/user/deposits             获取充值记录
GET    /api/v1/user/withdrawals          获取提现记录
```

**Bot 授权管理：**
```
GET    /api/v1/user/bot/authorizations   获取已授权的 Bot 列表
POST   /api/v1/user/bot/authorize        授权 Bot
PUT    /api/v1/user/bot/authorize/{id}   修改授权
DELETE /api/v1/user/bot/authorize/{id}   撤销授权
```

**Vault 管理：**
```
GET    /api/v1/user/vaults               获取 Vault 存款
POST   /api/v1/user/vault/deposit        存入 Vault
POST   /api/v1/user/vault/withdraw       提出 Vault
```
